# 策略详细分析 — ML 涨停预测信号融合

> 本篇记录将机器学习涨停预测模型融入交易系统的完整探索过程

---

## 一、ML 模型概述

### 1.1 模型架构

涨停预测系统基于三模型融合：
- LightGBM、CatBoost、XGBoost 各训练 4 个窗口（1d/5d/10d/20d）
- 共 12 个子模型，输出加权融合为 `composite_score`（0-1）
- 每日收盘后预测次日 Top 50 最可能涨停的股票

### 1.2 信号格式

```csv
vt_symbol,p_1d,p_5d,p_10d,p_20d,composite_score
605299.SSE,0.950,0.852,0.845,0.691,0.874
002082.SZSE,0.957,0.835,0.837,0.673,0.868
```

### 1.3 数据可用性

- 信号路径：`vnpy/examples/limit_up_prediction/signals/YYYY-MM-DD.csv`
- 可用区间：2026-01-05 ~ 2026-04-03（54 个交易日）
- 每日 Top 50 只股票

## 二、方案 B 尝试：仓位增强乘数（失败）

### 2.1 设计思路

不改变选股逻辑，仅在已选中的 A/A+ 股票上，根据 ML composite_score 加仓：
```
score >= 0.85 → ml_boost = 1.3x（强信号）
score >= 0.70 → ml_boost = 1.15x（中信号）
其他          → ml_boost = 1.0x（无影响）
```

### 2.2 失败原因：零交集

| 检查点 | A/A+ 股票数 | ML Top 50 | 重叠数 |
|--------|-----------|-----------|--------|
| 2026-01-05 | 182 | 50 | **0** |
| 2026-02-02 | 167 | 50 | **0** |
| 2026-03-02 | 155 | 50 | **2** |

**根本原因**：两个系统选股逻辑完全不同：
- 我们的系统：基本面驱动（估值、财务、行业舆情）→ 选出低估值蓝筹
- ML 模型：量价因子驱动（200 个技术指标）→ 选出短线动量股

这两个股票池几乎没有交集，方案 B 的乘数永远等于 1.0，完全无效。

### 2.3 关键发现

> **基本面选股和量价选股是两个正交的维度**。试图在一个维度上叠加另一个维度的信号，不如让它们各自独立运行、互相对冲。

## 三、方案 C 实施：独立 ML 动量子策略（成功）

### 3.1 架构设计

利用已有的 Ensemble 多策略框架，将 ML 信号作为独立子策略运行：

```
ml_tactical 组合
├── concentrated_v2（85%）— 基本面价值仓（主力）
└── ml_momentum（15%）   — ML 动量探索仓（卫星）
```

### 3.2 ml_momentum 子策略特点

| 参数 | 设置 | 理由 |
|------|------|------|
| 选股来源 | ML Top 50（score >= 0.70） | 绕过基本面评级，直接用 ML 信号 |
| 评级映射 | score >= 0.85 → A+, >= 0.70 → A | 让 ML 高分股通过买入过滤器 |
| 止损 | -5%（严格） | 动量股波动大，必须快速止损 |
| 时间止损 | 2 个月 | 短线逻辑，不长期持有 |
| 最大持仓 | 5 只 | 集中在最高概率的标的 |

### 3.3 资金分配（Regime 动态调整）

| 市场状态 | concentrated_v2 | ml_momentum |
|---------|----------------|-------------|
| risk_on | 75% | **25%** |
| neutral | 85% | 15% |
| risk_off | 90% | **10%** |

牛市时给 ML 动量仓更多资金（追涨停机会多），熊市时收缩（动量策略在熊市容易踩雷）。

## 四、回测结果（2026 Q1 短期验证）

### 4.1 核心指标对比

| 策略 | 总收益率 | 超额收益 | 最大回撤 |
|------|---------|---------|---------|
| **ml_tactical (组合)** | **-0.23%** | **+3.83%** | **-4.67%** |
| concentrated_v2 (基准) | -5.91% | -1.85% | -5.91% |
| 沪深300 | -4.06% | — | — |

### 4.2 关键发现

1. **ML 动量仓提供了正向贡献**：在 concentrated_v2 亏损 -5.91% 的同期，ml_tactical 仅亏 -0.23%，ML 子策略的短线利润对冲了主策略的回撤。

2. **超额收益显著**：ml_tactical 跑赢沪深300 +3.83%，而同期 concentrated_v2 跑输 -1.85%。

3. **回撤大幅改善**：-4.67% vs -5.91%，降低了 1.24 个百分点。

4. **互补性验证**：基本面价值仓和 ML 动量仓的收益曲线呈低相关性，组合后波动率下降。

## 五、技术实现要点

### 5.1 代码格式转换

ML 信号使用 vnpy 格式（`600488.SSE`），交易系统使用纯数字（`600488`）：
```python
def _convert_vt_symbol(vt_symbol: str) -> str:
    return vt_symbol.split(".")[0]
```

### 5.2 信号匹配逻辑

月度检查点需要匹配最近的 ML 信号日（向前查找）：
```python
def get_ml_score_at_checkpoint(ml_signals, code, checkpoint):
    best_date = max(d for d in ml_signals if d <= checkpoint)
    return ml_signals[best_date].get(code, 0.0)
```

### 5.3 Ensemble 引擎中的 ML 选股

ml_momentum 子策略绕过基本面评级，直接用 ML 信号构建评级：
```python
if strategy.config.name == "ml_momentum":
    sub_ratings = _build_ml_ratings(ml_signals, cp, prices)
else:
    sub_ratings = ... # 正常基本面评级
```

## 六、局限性与未来方向

### 6.1 当前局限

- **样本量不足**：仅 54 个交易日（3 个月），统计显著性有限
- **未经历完整牛熊周期**：2026 Q1 整体偏弱，未验证牛市表现
- **ML 模型本身的衰减**：量价因子的预测力可能随时间衰减，需定期重训练

### 6.2 未来方向

| 方向 | 描述 | 前提条件 |
|------|------|---------|
| 延长回测 | 积累 6-12 个月 ML 信号后做完整验证 | 持续运行每日预测 |
| 参数调优 | 调整资金比例（10-30%）和 score 阈值 | 更多数据支撑 |
| 多窗口利用 | 分别利用 p_1d/p_5d/p_10d/p_20d 做不同持仓周期 | 需要周频检查点 |
| 模型迭代 | 将基本面因子加入 ML 特征，缩小两个系统的"认知鸿沟" | 需修改训练代码 |

---

## 七、核心结论

> **基本面选股和量价选股是两个正交维度，不应该在同一个策略内混合，而应该作为独立子策略通过 Ensemble 框架协同运行。** ML 动量仓的短线利润可以有效对冲基本面价值仓的短期回撤，初步验证了"价值 + 动量"双因子组合的互补性。
