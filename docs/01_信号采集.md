# 01 - 信号采集

## 概述

系统通过 7 个适配器（Adapter）从不同子项目采集信号，统一写入 `stock_signals` 表。每个适配器继承 `BaseAdapter`，实现 `run()` 方法，输出标准格式：

```python
{
    "code": "000858",         # 股票代码
    "date": "2026-03-30",     # 分析日期
    "source": "screener",     # 信号来源
    "signal": "bullish",      # bullish / neutral / bearish
    "score": 82.4,            # 归一化 0-100
    "confidence": 82.4,       # 置信度 0-100
    "detail_json": {...}      # 各模型特有的详细数据
}
```

## 7 个信号源

### 1. Screener — 财务筛选（权重 25%）

**子项目**: `a-stock-screener`

**评分维度**（8 维度加权）：

| 维度 | 权重 | 关键指标 |
|------|------|---------|
| 增长质量 | 15% | 收入/利润 CAGR、正增长年占比 |
| 盈利持续性 | 20% | 毛利率均值及稳定性、ROE、ROIC |
| 资产负债质量 | 15% | 资产负债率、有息负债率、商誉占比 |
| 现金流质量 | 15% | 经营现金流/净利润、自由现金流正比例 |
| 资本配置 | 10% | 分红一致性、商誉变化趋势 |
| 抗风险能力 | 15% | 利润最大回撤、危机年（2020/2022）表现 |
| 竞争壁垒 | 10% | 毛利率稳定性及水平、ROE>15% 持续年数 |

**一票否决**: 净资产为负 → 直接淘汰

**信号映射**: 总分 ≥75 = bullish, 40~75 = neutral, <40 = bearish

---

### 2. Valuation — 估值分析（权重 30%）

**子项目**: `a-stock-valuation`

**实时模式**: 调用子项目的 4 模型估值（DCF、相对估值等），输出 weighted_gap（低估/高估幅度）

**历史回测模式**: 基于数据库历史数据计算 4 个估值指标：

| 模型 | 逻辑 |
|------|------|
| PE 估值 | PE_TTM 分档：<10 极度低估, <15 低估, 15~30 合理, >40 高估 |
| PB 估值 | PB vs 1.0 基准，ROE>15% 允许更高 PB |
| PEG 估值 | PE / 盈利增速：<1 低估, >2 高估 |
| PS 估值 | 市销率分档评估 |

**信号映射**: 4 模型均分 ≥65 = bullish, <35 = bearish

---

### 3. Buffett — 巴菲特模型（权重 20%）

**子项目**: `a-stock-analysis`

**评分维度**：

| 维度 | 分值 | 评判标准 |
|------|------|---------|
| 基本面质量 | 0-10 | ROE>15%, 负债率<50%, 净利率>15%, 流动比率>1.5, 盈利一致性 |
| 护城河 | 0-5 | ROE 持续性, 毛利率稳定性 |
| 管理层 | 0-2 | 分红记录（≥5年得 2 分） |
| 估值 | 0-5 | PE 历史分位数 + 安全边际 |

**信号映射**: 总分归一化到 0-100，≥70 = bullish, <40 = bearish

---

### 4. Munger — 芒格模型（权重 15%）

**子项目**: `a-stock-analysis`

**评分维度**：

| 维度 | 权重 | 关键指标 |
|------|------|---------|
| 护城河强度 | 35% | ROIC 持续性, 毛利率水平 + 稳定性 |
| 管理层质量 | 25% | FCF/净利润比率, 资产负债率, 分红记录 |
| 可预测性 | 25% | 收入增长稳定性, 净利率波动率, 经营现金流正比例 |
| 估值 | 15% | PE 分档, FCF 收益率 |

**信号映射**: 加权总分 ≥75 = bullish, <45 = bearish

---

### 5. Chan — 缠论技术面（权重 10%）

**子项目**: `a-stock-chan`

**分析流程**: K 线 → 合并 → 分型 → 笔 → 线段 → 中枢 → 走势类型 → 背驰 → 买卖点

**评分维度**：
- 买卖点信号（近 30 天内的买/卖点数量和置信度）
- 趋势状态（上升/下降/盘整）
- 背驰信号（底背驰/顶背驰）
- MACD 状态（金叉/死叉）
- 中枢位置（价格在中枢上方/内部/下方）

**信号映射**: 强烈买入/买入 = bullish, 卖出 = bearish, 关注/观望 = neutral

---

### 6. TrendRadar — 舆情分析

**子项目**: `TrendRadar`

**数据来源**: `finance_public_opinion.ai_analysis_results` 表的 AI 分析结果

**逻辑**: 匹配股票所属行业板块 → 读取板块利多/利空/中性判断 + 置信度

**信号映射**: 板块利多 = bullish, 板块利空 = bearish

---

### 7. Risk Manager — 风控管理

**子项目**: `a-stock-analysis`

**计算逻辑**:
- 日波动率 = 近 30 天收益率标准差
- 年化波动率 = 日波动率 × √252
- 波动率分位 = 当前波动率在历史中的百分位
- 仓位上限 = 基于波动率的 Kelly 公式推导

**输出**: 每只股票的建议仓位上限（%），供评级引擎使用
