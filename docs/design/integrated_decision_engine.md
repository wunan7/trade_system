## 2. 核心洞察：现有系统缺什么

已有完整的 **"选股→估值→择时→仓位"** 各环节独立工具，但它们之间是 **断裂** 的。用户需要手动串联、交叉比对、做最终决策。

**缺失的三个关键层：**

1. **决策融合层** — 没有系统把各模型的信号汇总成统一的"该不该买、买多少"的最终建议
2. **时间协同层** — 价值分析（screener/valuation）是静态的，缠论/舆情是动态的，没有"好公司+好价格+好时机"的交汇点检测
3. **持续跟踪层** — 跑完一次就完了，没有对候选池的持续监控、信号变化追踪、回测验证

## 4. 各层具体设计

### 4.1 第一层：统一数据层（已有，需小幅整合）

所有系统已共享 PostgreSQL `finance` 库。需要增加一张 **统一信号表**：

```sql
CREATE TABLE stock_signals (
    code        VARCHAR(10),
    date        DATE,
    source      VARCHAR(30),   -- 'screener'/'valuation'/'buffett'/'munger'/'damodaran'/'chan'/'trendradar'
    signal      VARCHAR(10),   -- 'bullish'/'bearish'/'neutral'
    score       FLOAT,         -- 各系统的归一化评分 0-100
    confidence  FLOAT,         -- 置信度
    detail_json JSONB,         -- 各系统的详细输出
    PRIMARY KEY (code, date, source)
);
```

### 4.2 第二层：多维分析引擎（保留 & 裁剪）

**保留的 6 个核心模块：**

| 模块 | 来源系统 | 角色 | 理由 |
|------|---------|------|------|
| **财报质量筛选** | a-stock-screener | 第一道门 | 8维评分体系成熟，覆盖全A，是最好的"好公司"过滤器 |
| **多模型估值** | a-stock-valuation | 价格锚定 | 4种估值方法互补，加权共识比单一方法稳健 |
| **大师视角** | Buffett + Munger | 定性研判 | LLM加持的多视角分析，提供人类可读的投资叙事 |
| **缠论择时** | a-stock-chan | 买卖时机 | 唯一的技术分析来源，买卖点信号无可替代 |
| **舆情催化** | TrendRadar | 催化剂检测 | 板块信号+情绪分析，是基本面之外的独立信息源 |
| **仓位管理** | Risk Manager | 风控 | 唯一的仓位控制工具 |

**建议降低优先级的 2 个模块：**

| 模块 | 理由 |
|------|------|
| **Damodaran 模型** | 与 a-stock-valuation 的增强DCF高度重叠（都是FCFF折现），且 a-stock-valuation 已包含情景分析更完善。可保留为可选视角，但不进入核心链路 |
| **vnpy 涨停预测** | 投资哲学与整体价值投资框架冲突。涨停追逐是短线投机，与 Buffett/Munger 的长期持有理念矛盾。可作为独立工具保留，但不建议纳入主流程 |

### 4.3 第三层：决策融合引擎（核心新增）

#### 4.3.1 综合评级系统（多模型投票）

```
最终评级 = 加权投票

┌────────────────┬────────┬──────────────────────────────┐
│     投票者      │  权重  │          投票内容             │
├────────────────┼────────┼──────────────────────────────┤
│ screener 评分   │  25%   │ 财务质量分（0-100 归一化）     │
│ valuation 信号  │  30%   │ 估值偏差（低估/高估/中性）     │
│ Buffett 信号    │  20%   │ 价值投资视角（bullish/neutral/bearish）│
│ Munger 信号     │  15%   │ 质量优先视角                  │
│ 缠论信号        │  10%   │ 技术面加减分                  │
└────────────────┴────────┴──────────────────────────────┘

综合评级：
  A+ : 所有模型一致看多（极稀缺，全A可能<20只）
  A  : 多数模型看多，无看空
  B  : 基本面优秀但估值中性，或估值低估但基本面一般
  C  : 信号混杂
  D  : 多数模型看空
```

#### 4.3.2 三重共振检测器（最高价值功能）

检测 **价值+技术+情绪** 三个独立维度同时发出信号的时刻：

```
黄金买点 = 以下三条同时满足：

  ① 价值面认可：valuation 信号=bullish 或 Buffett/Munger 中至少一个=bullish
  ② 技术面确认：缠论出现一类/二类买点（5天内），且MACD金叉
  ③ 情绪面催化：TrendRadar 板块信号=利多，置信度>60%

→ 三重共振触发时自动推送预警
```

同理有 **三重卖出预警**（估值高估 + 缠论卖点 + 板块利空）。

#### 4.3.3 仓位计算器（升级版 Risk Manager）

在现有 Risk Manager（波动率+相关性）基础上，叠加综合评级：

```
最终仓位 = Risk Manager 仓位上限 × 信号强度系数

信号强度系数：
  A+ 评级 → 1.0（满仓位配置）
  A  评级 → 0.8
  B  评级 → 0.5
  C  评级 → 0.3（试探仓）
```

### 4.4 第四层：输出与监控

#### 4.4.1 每日综合简报

复用 TrendRadar 的推送通道，每日推送：

```
═══════════════════════════════════════
  每日投资决策简报 2026-03-24
═══════════════════════════════════════

【三重共振预警】
  - 600519 贵州茅台：估值低估(-19%) + 缠论二类买点(3天前) + 白酒板块利多
    → 建议仓位：15%（Risk Manager上限20% × 信号系数0.8）

【综合评级变动】
  - 000858 五粮液：B → A（screener评分升至82，valuation转为bullish）
  - 002714 牧原股份：A → C（周期性利润下滑，Munger信号转bearish）

【候选池监控】
  A+级（3只）：...
  A 级（15只）：...

【风控提醒】
  - 持仓中 002415 海康威视 出现缠论一类卖点，建议关注
═══════════════════════════════════════
```

#### 4.4.2 回测验证系统

定期回测各模型和综合评级的历史表现，输出：
- 各模型信号的历史胜率
- 三重共振信号的历史收益率
- 综合评级的 Alpha 贡献分解

---

## 6. 核心升级总结

1. **多模型投票**：5个独立视角加权投票，比任何单一大师模型都稳健
2. **三重共振**：价值×技术×情绪同时确认，大幅提升信号可靠性
3. **闭环追踪**：从筛选→评级→择时→仓位→推送→回测，形成完整闭环
