## 历史信号预计算

### 为什么需要预计算

实时评级通过 7 个适配器调用子项目实时分析，但回测需要知道"历史某天的信号是什么"。如果每次回测都重新计算，速度极慢且可能引入前视偏差。

预计算的解决方案：
1. 对每个月度检查点，使用**只在该日期之前可用的数据**计算信号
2. 结果存入 `stock_signals` 表（复用现有 schema）
3. 回测时直接查表，与实时评级使用**完全相同的评分公式**

### 5 个历史信号计算器

| 信号源 | 实现方式 | 数据来源 |
|--------|---------|---------|
| screener | 复刻 8 维度评分 | financial_summary（仅用 checkpoint 前的年报） |
| valuation | PE/PB/PEG/PS 四模型 | stock_daily（价格）+ financial_summary（EPS/BPS） |
| buffett | 基本面 + 护城河 + 管理层 + 估值 | financial_summary + financial_balance |
| munger | 护城河 + 管理层 + 可预测性 + 估值 | financial_summary + financial_cashflow |
| chan | **直接调用 a-stock-chan** | stock_daily（K 线截止到 checkpoint） |

### 前视偏差防护

- `_filter_annual_only(records, checkpoint)`: 只使用 `report_date <= checkpoint` 的年报
- chan 分析: `analyze_one_from_db(code, end_date=checkpoint)` 截断 K 线数据
- `days_ago()` 使用 K 线最后日期作为参考（而非系统当前时间）

### 执行

```bash
# 完整预计算（含缠论，约数小时）
python -m src precompute --start 2023-03-27 --end 2026-03-27

# 跳过缠论（快速，约 10 分钟）
python -m src precompute --start 2023-03-27 --end 2026-03-27 --skip-chan

# 只计算某个信号源
python -m src precompute --source valuation --start 2023-03-27 --end 2026-03-27
```

支持**断点续传**：自动跳过 `stock_signals` 中已有的 (code, date, source) 记录。

## 策略配置

### 策略文件

每个策略是一个 JSON 文件，存放在 `strategies/` 目录：

```
strategies/
├── default.json       # 基准策略
├── aggressive.json    # 激进策略
├── conservative.json  # 保守策略
└── my_strategy.json   # 自定义（复制修改即可）
```

### 可配置参数

```json
{
  "name": "default",
  "description": "基准策略：A级持仓，-10%止损，不止盈",

  // 信号权重（影响评级结果）
  "rating_weights": {
    "screener": 0.25, "valuation": 0.30,
    "buffett": 0.20, "munger": 0.15, "chan": 0.10
  },
  // 评级阈值
  "rating_thresholds": { "A+": 85, "A": 70, "B": 55, "C": 40 },

  // 买入条件
  "buy_ratings": ["A+", "A"],           // 只买这些评级
  "position_sizes": { "A+": 0.20, "A": 0.15 },  // 基础仓位
  "resonance_boost": 1.5,              // 三重共振加仓倍数
  "high_score_boost": 1.2,             // 高分加仓倍数
  "high_score_threshold": 80,          // 高分阈值
  "max_single_position": 0.25,         // 单只上限
  "max_holdings": 10,                  // 最多持有

  // 卖出条件
  "stop_loss_pct": -0.10,              // 止损线（null=不止损）
  "take_profit_pct": null,             // 止盈线（null=不止盈）
  "sell_on_downgrade_to": ["C", "D"],  // 评级降至这些级别时卖出

  // 执行参数
  "checkpoint_frequency": "monthly",
  "initial_capital": 100000,
  "min_rebalance_diff": 1000,
  "lot_size": 100
}
```

### 预置策略对比

| 参数 | default | aggressive | conservative |
|------|---------|-----------|-------------|
| 买入评级 | A+/A | A+/A/B | 仅 A+ |
| 止损 | -10% | -15% | -5% |
| 止盈 | 不止盈 | +30% | +15% |
| 缠论权重 | 10% | 30% | 5% |
| 估值权重 | 30% | 25% | 35% |
| 最多持仓 | 10 只 | 15 只 | 5 只 |
| 单只上限 | 25% | 30% | 15% |
| 降级卖出 | C/D | D | A 以下 |

## 回测执行

### 回测流程

```
for checkpoint in 月度检查点:
    1. 从 stock_signals 读取该日预计算信号
    2. 用策略配置的权重和阈值计算评级
    3. 执行调仓：
       - 止盈止损检查
       - 降级卖出
       - 买入新达标股票（按评分排序）
       - 加仓/减仓已持仓股票
    4. 记录净值
```

### 执行命令

```bash
# 用默认策略回测 3 年
python -m src advanced-backtest --strategy default --start 2023-03-27 --end 2026-03-27

# 用激进策略回测
python -m src advanced-backtest --strategy aggressive --start 2023-03-27 --end 2026-03-27

# 列出所有可用策略
python -m src list-strategies
```

### 报告输出

文件名格式：`reports/backtest_{策略名}_{生成时间}.md`

报告结构：
- **第一部分：回测结果** — 收益统计表（总收益率、年化、超额、最大回撤、胜率）+ 净值曲线
- **第二部分：详细交易策略** — 从策略 JSON 动态生成（权重、阈值、买卖规则、风控参数）

### 回测结果示例（2023-03-27 ~ 2026-03-27）

| 策略 | 总收益率 | 年化 | 超额收益 | 最大回撤 |
|------|---------|------|---------|---------|
| default | +21.58% | +6.72% | +3.73% | -16.99% |
| aggressive | +1.18% | +0.39% | -16.67% | -22.72% |
| conservative | -0.41% | -0.14% | -18.26% | -22.75% |
