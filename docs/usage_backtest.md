# 回测验证

## 概述

`python -m src backtest` 对 `stock_signals` 和 `integrated_ratings` 中的历史信号进行前向收益率分析，衡量各模型的预测能力。

回测需要多日累积数据才有统计意义。建议连续运行 `collect + rate` 至少 20 个交易日后再查看回测结果。

## 三大评估维度

### 1. 各模型信号胜率

对每个 source 的 bullish/bearish 信号，统计信号发出后 N 个交易日的涨跌情况。

**输出示例**:

| source | signal | count | 胜率5d | 均值5d | 胜率10d | 均值10d | 胜率20d | 均值20d |
|--------|--------|-------|--------|--------|---------|---------|---------|---------|
| screener | bullish | 1200 | 58.3% | +1.2% | 61.2% | +2.1% | 64.5% | +3.8% |
| valuation | bullish | 980 | 55.1% | +0.8% | 58.7% | +1.5% | 62.3% | +2.9% |
| chan | bullish | 850 | 53.2% | +0.5% | 52.8% | +0.3% | 51.5% | +0.1% |

**解读**: 胜率 > 50% 表示该信号有预测价值；均值 > 0 表示跟随信号可盈利。

### 2. 三重共振收益率

评估三重共振信号的持仓收益，使用 T+1 开盘价入场（防止前视偏差）。

**输出示例**:

| 持仓天数 | 样本数 | 胜率 | 平均收益 | 中位收益 |
|---------|--------|------|---------|---------|
| 5d | 28 | 67.9% | +2.35% | +1.82% |
| 10d | 28 | 71.4% | +4.12% | +3.55% |
| 20d | 28 | 75.0% | +6.89% | +5.43% |

### 3. 综合评级 Alpha 分解

验证评级系统的有效性：高评级股票是否真的跑赢低评级和市场平均。

**输出示例**:

| 评级 | count | 均值5d | Alpha5d | 均值10d | Alpha10d |
|------|-------|--------|---------|---------|----------|
| A | 42 | +2.1% | +1.5% | +3.5% | +2.8% |
| B | 189 | +0.8% | +0.2% | +1.2% | +0.5% |
| C | 683 | +0.4% | -0.2% | +0.5% | -0.2% |
| D | 4098 | +0.2% | -0.4% | +0.1% | -0.6% |
| market | 5012 | +0.6% | 0.0% | +0.7% | 0.0% |

**解读**: Alpha > 0 表示该评级组跑赢市场平均。期望看到 A > B > C > D 的单调递减。

## 常用操作

```bash
# 回测当天信号（输出到终端）
python -m src backtest --dry-run

# 回测日期范围
python -m src backtest --start 2026-03-01 --end 2026-03-29

# 自定义持仓天数
python -m src backtest --holding-days 5,10,20,60

# 保存报告到 reports/ 目录
python -m src backtest --start 2026-03-01 --end 2026-03-29
# 输出: reports/backtest_2026-03-29.md
```

## 数据积累建议

| 积累天数 | 可靠性 | 说明 |
|---------|--------|------|
| 1-5 天 | 不可靠 | 样本太少，随机波动大 |
| 10-20 天 | 初步参考 | 趋势可见，但置信度不足 |
| 20-60 天 | 可用 | 各模型胜率差异开始显现 |
| 60+ 天 | 较可靠 | 可作为权重调优依据 |

## 根据回测调优权重

当回测数据充分后，若发现某个 source 的胜率持续偏低，可在 `src/config.py` 中调低其权重：

```python
# 例：valuation 胜率高，chan 胜率偏低
RATING_WEIGHTS = {
    "screener": 0.25,
    "valuation": 0.35,   # 提高
    "buffett": 0.20,
    "munger": 0.15,
    "chan": 0.05,         # 降低
}
```

修改权重后重新运行 `python -m src rate` 即可生效。
